# DX.Comply – Release & Milestone Plan

## Goal
Deliver a stable MVP (v1.0) within 6-8 weeks, followed by incremental releases.

## Milestones
### M1 – Core (Week 1–2)
- ProjectScanner (.dproj)
- FileScanner (output scan)
- HashService (SHA-256)
- CycloneDX Writer (JSON)

### M2 – IDE (Week 3)
- ToolsAPI Wizard
- Menu / toolbar
- Logging & error handling

### M3 – CLI (Week 4)
- dxcomply CLI
- CI documentation (GitHub Actions / GitLab CI)

### M4 – Stabilisation (Week 5)
- Performance optimisation
- Schema validation
- Edge cases

### M5 – Release (Week 6)
- Signed binaries
- Installer
- README / docs
- Example projects

## Post-v1
- v1.1: SPDX, HTML report, SBOM diff
- v2.0: Optional CVE matching, licence heuristics, policy checks
